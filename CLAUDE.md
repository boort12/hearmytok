# CLAUDE.md — HearMyTok

TikTok music analytics platform for A&Rs and artists to discover viral songs and emerging artists. Currently frontend-only with mock data. Backend will be built separately.

---

## Current State

**Frontend only.** No backend. All data comes from `frontend/lib/mockData.ts`.

---

## Project Layout

```
hearmytok/
├── frontend/        — Next.js 16 app (the only thing running)
│   ├── app/
│   │   ├── page.tsx              — Dashboard: trending track cards
│   │   ├── charts/page.tsx       — Sortable top-100 table
│   │   └── tracks/[id]/page.tsx  — Track detail + growth chart
│   ├── components/
│   │   ├── TrendCard.tsx         — Reusable track card
│   │   └── ui/                   — shadcn components
│   └── lib/
│       ├── api.ts                — Type definitions + API client (unused until backend exists)
│       └── mockData.ts           — Dummy data powering all 3 pages
└── DEVLOG.md        — Full build history
```

---

## Tech Stack

- **Next.js 16.2** — routing, SSR. **Breaking changes from older versions — check `node_modules/next/dist/docs/` before writing any Next.js code.**
- **React 19**
- **TypeScript 5**
- **Tailwind CSS 4**
- **shadcn/ui** (`@base-ui/react`) — Card, Badge
- **Recharts 3** — growth line chart on track detail page

---

## Running

```bash
cd frontend
npm install
npm run dev    # → http://localhost:3000
```

No backend, no env vars, no database needed.

---

## Pages

| Route | File | Description |
|---|---|---|
| `/` | `app/page.tsx` | Dashboard — grid of trending track cards |
| `/charts` | `app/charts/page.tsx` | Sortable table: rank / growth % / video count |
| `/tracks/[id]` | `app/tracks/[id]/page.tsx` | Track stats + Recharts growth line chart |

---

## Mock Data

All data is in [frontend/lib/mockData.ts](frontend/lib/mockData.ts). Two exports:

- `mockArtists` — 8 artists with follower counts
- `mockTracks` — 10 tracks each with a `latest_snapshot` and 10 historical `snapshots` (generated by `makeSnapshots`)

When the backend is ready, replace mock imports in each page with calls from `lib/api.ts`.

---

## Data Shape (matches future backend schema)

```ts
Track       — id, title, artist_name, cover_url, tiktok_sound_id, latest_snapshot, snapshots[]
Artist      — id, name, tiktok_followers, tiktok_author_id, spotify_id
TrendSnapshot — video_count, play_count, growth_rate, rank, timestamp
```

---

## Key Notes

- Next.js 16 has breaking changes vs 14/15 — always check the bundled docs first.
- The `lib/api.ts` functions exist as a typed placeholder for when the backend is live. They are not called anywhere yet.
- Mock data snapshots are generated with `makeSnapshots(baseVideos, baseGrowth)` — 10 data points spaced 6 hours apart.
